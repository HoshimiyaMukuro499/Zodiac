# Godot 4 初学者入门教程

这是一份专为完全零基础新手准备的 Godot 4 入门指南，涵盖核心概念、脚本语法、项目结构和学习资源。


## 一、Godot 是什么？

Godot 是一个**完全免费且开源**的游戏引擎，支持 2D 和 3D 游戏开发。它的特别之处在于：

- **完全免费**：没有版税、没有订阅费
- **轻量级**：下载包很小，运行流畅
- **独立场景系统**：用节点（Node）搭积木的方式来组织游戏


## 二、核心概念：节点与场景

这是 Godot 最重要的设计理念，理解这两样东西就等于理解了 Godot 的一半。

### 2.1 节点（Node）

节点是 Godot 的**基本建筑块**。每个节点都有一个特定的功能：

| 节点类型 | 作用 |
|---------|------|
| `Node2D` | 2D游戏对象的基类 |
| `Sprite2D` | 显示图片 |
| `CharacterBody2D` | 可控制的角色 |
| `CollisionShape2D` | 添加碰撞检测 |
| `Timer` | 计时器，延迟执行 |
| `AudioStreamPlayer` | 播放声音 |

### 2.2 场景（Scene）

场景就是**一组节点的组合**，可以保存为单独的文件（`.tscn`）。一个游戏通常由多个场景组成：
- `Player.tscn` - 玩家角色
- `Enemy.tscn` - 敌人
- `Main.tscn` - 主游戏界面
- `Menu.tscn` - 开始菜单

**关键操作：实例化（Instantiate）**
```gdscript
# 从文件加载场景并创建实例
var enemy = preload("res://enemy.tscn").instantiate()
add_child(enemy)  # 添加到当前场景
```


### 2.3 获取节点

在脚本中获取节点引用的方法：
```gdscript
extends Node2D

# 方法1：get_node()
var player = get_node("Player")

# 方法2：$ 简写（推荐）
var player = $Player

# 方法3：确保节点在 _ready 时已存在
@onready var animation = $AnimationPlayer
```



## 三、GDScript 基础语法

GDScript 是 Godot 的专属脚本语言，语法类似 Python，非常易学。

### 3.1 变量

```gdscript
# 基本类型
var score = 0          # 整数
var speed = 150.5      # 浮点数
var name = "Player"    # 字符串
var is_alive = true    # 布尔值

# 指定类型（可选但推荐）
var health: int = 100
var position: Vector2 = Vector2(0, 0)

# 常量
const GRAVITY = 9.8

# 导出变量（在编辑器中可见可调）
@export var jump_force: float = 400.0
```

### 3.2 函数

```gdscript
# 基本函数
func take_damage(amount: int) -> void:
    health -= amount
    if health <= 0:
        die()

# 带返回值的函数
func get_health_percentage() -> float:
    return float(health) / max_health * 100
```

### 3.3 生命周期函数（最重要！）

```gdscript
extends CharacterBody2D

# 当节点进入场景树时调用（只调用一次）
func _ready():
    print("游戏开始了！")
    position = Vector2(100, 200)

# 每帧调用（用于持续检测输入、更新状态）
func _process(delta):
    # delta 是上一帧花费的时间（秒）
    # 用 delta 让移动速度与帧率无关
    position.x += speed * delta

# 物理帧调用（用于物理运动，如角色移动）
func _physics_process(delta):
    # 配合 move_and_slide() 使用
    velocity.y += gravity * delta
    move_and_slide()
```


### 3.4 条件判断与循环

```gdscript
# if 语句
if health <= 0:
    die()
elif health < 30:
    play_hurt_animation()
else:
    pass  # 什么都不做

# for 循环
for i in range(5):
    print(i)  # 输出 0,1,2,3,4

# 遍历数组
var items = ["sword", "shield", "potion"]
for item in items:
    print(item)

# while 循环
var count = 0
while count < 10:
    count += 1
```

### 3.5 信号（Signal）——节点间通信

Godot 的特色功能，用于节点之间松耦合地通信：

```gdscript
# 在一个节点中定义信号
signal health_depleted

# 发送信号
func take_damage():
    health -= 10
    if health <= 0:
        health_depleted.emit()  # 发出信号

# 在另一个节点中监听信号
func _ready():
    $Player.health_depleted.connect(game_over)

func game_over():
    print("游戏结束")
```

### 3.6 常用运算符

```gdscript
# 算术运算
var result = (5 + 3) * 2  # 16

# 比较运算
if score > high_score: pass
if player_name == "Hero": pass

# 逻辑运算
if is_alive and has_key:
    open_door()

# 三元运算符

# 类型转换
var number = int("123")   # 字符串转整数
var text = str(456)       # 整数转字符串
```


## 四、项目文件结构

一个典型的 Godot 项目目录如下：

```
MyGame/                      # 项目根目录
├── project.godot            # 项目配置文件（自动生成，不要手动改）
├── scenes/                  # 场景文件
│   ├── main.tscn
│   ├── player.tscn
│   └── enemy.tscn
├── scripts/                 # 脚本文件
│   ├── player.gd
│   └── enemy.gd
├── assets/                  # 资源文件
│   ├── images/
│   ├── sounds/
│   └── fonts/
└── shaders/                 # 着色器文件（可选）
```

**重要路径规则**：`res://` 表示项目根目录
```gdscript
var texture = load("res://assets/images/player.png")
var scene = preload("res://scenes/bullet.tscn")
```


## 五、运行方式

### 5.1 在编辑器中运行

1. 打开要测试的场景
2. 点击右上角的 **"运行当前场景"** 按钮（或按 **F6**）
3. 或设置主场景后点击 **"运行项目"**（按 **F5**）

### 5.2 设置主场景

项目启动时首先运行的场景：
- 菜单栏 → **项目** → **项目设置** → **常规** → **应用程序** → **运行** → **主场景**
- 或直接在文件系统中右键点击场景文件，选择 **"设为主场景"**

### 5.3 导出游戏

菜单栏 → **项目** → **导出** → 添加导出模板 → 选择平台（Windows/macOS/Linux/Web）

Godot 支持一键导出到多个平台，无需额外配置。

## 八、新手常见错误与避坑

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `Invalid call. Nonexistent function` | 函数名拼写错误 | 检查大小写，GDScript 区分大小写 |
| `Cannot call method on null instance` | 节点路径写错 | 检查 `$` 后面的路径与实际场景树是否一致 |
| 移动速度时快时慢 | 忘了乘以 `delta` | 在 `_process` 中始终乘以 `delta` |
| 碰撞没反应 | 没设置碰撞层/掩码 | 检查两个物体的碰撞层和掩码是否匹配 |
| 信号连不上 | 函数名与连接的名称不一致 | 信号连接窗口中的"Receiver Method"必须与实际函数同名 |
