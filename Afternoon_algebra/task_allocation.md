## 一、四角色一句话定位

| 角色 | 一句话定位 |
|------|-----------|
| **A** | 游戏骨架：棋盘、移动、碰撞、回合、胜负 —— 让游戏“能玩下去” |
| **B** | 特殊机制：6种球的全部特殊规则（变色、随从、推挤、分步、干扰、增益） |
| **C** | 界面交互：画棋盘、点选、按钮、动画、信息显示 —— 让玩家“看得见、点得动” |
| **D** | 规则兜底：边界情况、并发冲突、测试用例、Bug修复 —— 让游戏“不出错” |
D原则上是debug与调试岗，但是必要时也要协助B/C进行代码编写

## 二、按周分工（每人独立任务清单，无交叉依赖）

### 第一周：Demo（白球 + 蓝球，可对局）

#### A - 骨架
- [ ] 实现六边形169格点数据结构（立方体坐标或偏移坐标）
- [ ] 实现边界判定：`D = max(|x|,|y|,|x-y|) >= 8` 即出界
- [ ] 实现**逐格移动**（步数=力度，一格一格走）
- [ ] 实现**弹性碰撞**：进入有弹珠的格子 → 两者速度交换（当前球停，对方继续走剩余步数）
- [ ] 实现出界死亡检测（每步后检查）
- [ ] 实现回合切换 + 胜负判定（一方弹珠数=0）

#### B - 基础棋子
白球 + 蓝球
- [ ] 白球：方向+力度1~5，移动逻辑
- [ ] 蓝球：移动前在“不共线的4个邻格”中选2个空位生成随从
- [ ] 蓝球随从：同方向同力度移动，移动后消失
- [ ] 蓝球死亡：自身出界 **或 任意随从出界** → 蓝球死

#### C - UI
- [ ] 绘制六边形棋盘（169个格点可用占位符点或简单六边形）
- [ ] 绘制双方弹珠（颜色用纯色圆/圆点区分）
- [ ] 实现玩家操作流程：点击己方弹珠 → 选方向 → 选力度/步数 → 执行
- [ ] 显示当前轮到谁（红/蓝）
- [ ] 显示基本提示（选中球、剩余步数等）

#### D - 支撑与测试
- [ ] 搭建一个**手动测试场景**（预设几个坐标的弹珠，可快速验证碰撞）
- [ ] 验证边界点：`D=7` 安全，`D=8` 出界
- [ ] 验证白球碰撞：友方白球碰撞时+1剩余步数
- [ ] 验证蓝球随从生成：优先取移动方向两侧，被占则随机选剩余空位
- [ ] 记录并跟踪第一周的Bug清单

> ✅ 第一周末：**红蓝对战Demo**，白球+蓝球可完整对局，赢家能正常结束游戏。

### 第二周：6种球全部补齐

#### A - 补齐移动/碰撞骨架
- [ ] 确保黑球“强制移动敌方”走相同的碰撞逻辑（复用第一周代码）
- [ ] 确保黄球移动（力度1~5）和白球一致
- [ ] 协助B完成剩余4种球的物理法则
#### B - 剩余4种球 + 特殊交互
- [ ] **绿球**：移动后，相邻6格同时向远离方向推1格（目标格被占则失败不推，出界则死亡）
- [ ] **红球**：步数1~4，每步可选不同方向
- [ ] **黑球**：选中敌方，选主方向，系统随机偏±60°，随机步数{2,3}，强制移动
- [ ] **黄球**：死亡时选友方（非白/非黄）永久增益，不同球不同效果
- [ ] **白球变色**：己方其他颜色死亡时立即变色（黄球死亡不触发），多个白球优先未变色的

#### C - UI升级
- [ ] 红球专用UI：步数滑块 + 每步方向依次选择
- [ ] 黑球专用UI：选中敌方弹珠 + 选择大致方向
- [ ] 黄球增益显示：在弹珠上标注+1/+2等
- [ ] 绿球推挤动画（可选）

#### D - 复杂性兜底
- [ ] 实现**绿球同时推挤**（不能串行推，否则顺序影响结果）
- [ ] 处理**多人同时死亡时白球变色顺序**（按结算顺序，一次只变一个）
- [ ] 处理蓝球“无空位生成随从”时不触发随从出界死亡
- [ ] 处理黑球移动过程中与其它球的碰撞（完全复用A的碰撞函数）
- [ ] 编写6种球的交叉测试用例（例如黄球给蓝球增益后随从不再杀蓝球）

> ✅ 第二周末：6种球全部可玩，所有特殊规则可用。
